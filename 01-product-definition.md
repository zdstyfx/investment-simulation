# 01 · 产品定义

## 1. 一句话定位

一场让参与者同时体验「创业者」与「投资人」双重身份的半日沉浸式工作坊,系统是其中的虚拟投资交易平台。

## 2. 目标

1. 让小组在半天内把零散想法打磨成可路演的方案。
2. 通过虚拟投资博弈,培养商业判断力和风险决策能力。
3. 为企业(韶音)发掘优秀项目与人才。

## 3. 角色

| 角色 | 登录方式 | 能做什么 |
|---|---|---|
| **主办方 admin** | 固定内部账号(预置硬账号,不开放注册) | 创建/配置场次、控制回合与投资窗口、录入评审倍率、一键结算、揭榜、查看全量数据 |
| **小组 group** | 自助注册(用户名 + 密码) | 加入场次、命名队伍、看自己余额、在窗口开启时投资当前路演组、看自己的投资记录、揭榜后看公开榜单 |
| **大屏 screen** | 免登录只读 | 显示当前路演组、倒计时;揭榜时展示融资榜 / 收益榜 |

> 关键:小组只能命名**自己的队伍**;场次由主办方命名。投资金额对其他小组**保密**。

## 4. 场次(session)

系统以"场次"为最外层隔离单元。主办方可开多个场次,每个场次:

- 有独立的名称(主办方命名)与**场次码**(小组凭码加入)
- 有独立的一套配置(小组数量、初始资金、各项规则,见 `06-config-parameters.md`)
- 有独立的小组、投资数据、倍率、榜单

小组账号注册后,在场次列表中选择并凭场次码加入对应场次。正式赛前的组内预演/汇报,就开一个独立的练习场次即可,数据与正式赛分开。

## 5. 核心机制:虚拟投资系统

每个小组在某场次中初始获得一笔**虚拟资金**(默认 100 万,可配置),并拥有双重身份:

- **创业者**:上台路演,吸引其他组投资,追求自己项目"融资额"最高。
- **投资人**:把资金投给看好的其他组,追求"投资回报率"最高。

机制要点:

- 投资金额**不公开**(防跟风/羊群效应),服务端强制隔离。
- 仅融资额排名**前 N 名**(默认前 3)享受评审倍率加成;其余按基础倍率(默认 1x)结算。
- 允许保留资金不投(保守策略合法,可配置)。
- 不能投自己组(可配置)。

详见 `04-business-logic.md` 的回合流程与结算算法。

## 6. 命题方向

小组须在主办方设定的命题方向中选题(默认三个:生产力硬件 / 运动健康硬件 / AI + 新型硬件),方向列表与"是否强制选题"均可配置。

## 7. 奖项

| 奖项 | 评选依据 | 可开关 |
|---|---|---|
| **最会融资奖** | 项目融资总额最高的小组 | 是 |
| **最会投资奖** | 投资回报率(ROI)最高的小组 | 是 |
| **韶音特别关注奖** | 主办方综合判断(手动指定) | 是 |

> 只设这两类量化奖 + 一个手动奖,**不计算"总资产"合并排名**。
