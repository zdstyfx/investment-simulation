# 想法开张日 · 虚拟投资工作坊系统 — 开发交接文档

> 清华 X-Lab × 韶音 创新创业工作坊配套系统。本目录是交给 coding agent 的完整开发说明。
> 所有正文为中文,代码标识符(表名、字段、接口、配置键)统一用英文 snake_case,方便直接落到代码里。

## 这是什么

一个**由主办方控制节奏的、回合制虚拟投资交易系统**。参与小组在活动中同时是"创业者"(被投资)和"投资人"(投别人),通过一整套投融资模拟体验创业博弈。系统是小体量、单活动、不需要高并发的 Web 应用,核心难点只有三个:

1. **回合状态机** —— 主办方控制"当前哪组路演、投资窗口开/关",所有客户端跟随这个全局状态。
2. **投资金额保密** —— 任何小组都不能看到别组投了多少,必须在服务端强制隔离。
3. **一键结算** —— 活动尾声按融资榜 + 评审倍率算出最终估值、收益、排名。

## 推荐技术栈

为"易于在内网一台机器上部署、单进程、coding agent 友好"而选:

- **后端**:Node.js + Express + `better-sqlite3`(SQLite 单文件数据库)
- **前端**:React + Vite + Tailwind CSS(三个入口:`/admin` 主办方、`/play` 小组、`/screen` 大屏)
- **鉴权**:JWT(密钥放环境变量);bcrypt 做密码哈希
- **实时同步**:HTTP 轮询(小组端 / 大屏每 ~2 秒拉一次状态),不用 WebSocket
- **打包部署**:前端 `vite build` 出静态文件,由 Express 一并托管 —— 最终是**单个 Node 进程 + 一个 .sqlite 文件**,内网部署极简

## 运行与部署(国内场景)

- **开发**:`vite dev` + `node server.js`
- **活动当天(主力)**:在会场一台笔记本上 `node server.js`,大家连同一个 wifi/热点,浏览器访问 `http://<这台机器的局域网IP>:PORT`。**不依赖外网、不需要备案、延迟最低**,内网用纯 HTTP 即可。
- **需要公网可访问(联调/备用)**:用阿里云/腾讯云的**香港**轻量服务器(免 ICP 备案,可绑域名 + HTTPS,到大陆延迟低),用 Caddy/Nginx 反代加 HTTPS。
- **不要**用 Vercel/Railway/Render 等海外平台做正式部署:国内访问慢、易抽风。

## 角色与鉴权(重要)

- **小组账号**:自助注册,只需用户名 + 密码。注册登录后,看到主办方已开放的场次列表,凭"场次码"加入对应场次,并给自己小组命名。
- **主办方账号**:**不开放注册**。是一份固定的内部账号清单,部署时通过 seed 文件 / 环境变量种入 `admins` 表(密码 bcrypt 哈希),只在公司内部流传。
- **大屏**:免登录的只读公开页,只读取主办方控制的场次状态。

## 跨场次说明

主办方可以创建**多个独立场次**(如"7月3日正式场""组内预演场A"),数据按场次完全隔离。同一个小组账号可加入不同场次;练习与正式赛互不干扰。
